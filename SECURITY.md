# 安全政策

语言：中文 | [English](docs/en/SECURITY.md)

## 支持版本

当前维护重点是最新发布版本和 `master` 分支。报告问题时请尽量说明受影响版本、提交号、JDK、操作系统、部署方式和存储后端。

| 版本 | 支持状态 |
| --- | --- |
| `2.0.0-M27` | 支持 |
| `master` | 支持 |
| 更早版本 | 尽力协助 |

## 私密报告安全问题

请不要通过公开 issue、discussion、PR 或日志附件提交安全漏洞、利用细节、凭据、私钥、生产 DN、访问令牌或可识别用户数据。

推荐使用 GitHub Security Advisory 私密报告入口：

<https://github.com/dibyang/apacheds/security/advisories/new>

如果该入口不可用，请通过维护者认可的私密渠道联系项目维护者，并在公开仓库中只留下不包含利用细节的最小说明。

## 报告内容

安全报告请尽量包含：

- 受影响版本、提交号和部署方式。
- JDK、操作系统、网络暴露面和存储后端。
- 漏洞类型、影响范围和是否可能远程利用。
- 最小复现步骤或安全的概念验证。
- 日志、堆栈、配置片段和缓解建议。

请先脱敏所有密码、token、证书私钥、真实用户数据和生产环境专有信息。

## 处理流程

维护者会优先确认影响范围、复现条件和临时缓解方式。修复准备完成前，默认不公开利用细节。涉及依赖库、ApacheDS 上游或 JDBM 的问题，会在必要时协调对应项目。

## 非安全问题

普通 bug、性能问题、兼容性问题和文档问题请使用 GitHub bug 模板提交。无法判断是否属于安全问题时，优先按安全问题私密报告。
